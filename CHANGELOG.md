# Changelog

This repo uses [Changesets](https://github.com/changesets/changesets) for version management with fixed-versioning across `xci`, `@xci/server`, and `@xci/web`. Each release aggregates one or more changesets into a single coordinated version bump.

The authoritative per-package changelog is generated by Changesets on each release and lives at:

- [`packages/xci/CHANGELOG.md`](./packages/xci/CHANGELOG.md)
- [`packages/server/CHANGELOG.md`](./packages/server/CHANGELOG.md)
- [`packages/web/CHANGELOG.md`](./packages/web/CHANGELOG.md)

This root CHANGELOG summarises milestone-level events only.

---

## Milestones

### v2.0 — Remote CI (TBD — operator sets date on first publish)

**Packages:** `xci@2.0.0`, `@xci/server@2.0.0`, `@xci/web@2.0.0`

- **New: Agent registration & WebSocket protocol** — `xci --agent <url> --token <reg-token>` registers the machine as a persistent CI agent; TOFU credential stored locally; auto-reconnect with exponential backoff; heartbeat/drain/reconcile lifecycle.
- **New: Multi-tenant auth + org model** — signup/login/session with Argon2id; org creation; Owner/Member/Viewer roles; email-pinned invites; password reset; CSRF protection.
- **New: Task YAML DSL shared with server** — same alias/sequential/parallel/platform-block syntax as v1 CLI, validated server-side at save time; `${VAR}` resolved at dispatch time.
- **New: Secrets envelope encryption** — org-level secrets stored with AES-256-GCM (MEK → DEK per-org); agent-local `.xci/secrets.yml` wins on collision; MEK rotation endpoint with DEK re-wrap.
- **New: Dispatch pipeline + quota enforcement** — label-match dispatcher, in-memory queue with DB reconciliation, TaskRun state machine (queued → dispatched → running → terminal), per-org Free plan limits.
- **New: Log streaming + retention** — `log_chunk` frames streamed agent → server → browser; 8 KB chunk split; dual redaction (server + agent); Postgres persistence; daily retention cleanup; `.log` download endpoint.
- **New: Plugin system (GitHub + Perforce webhooks)** — `TriggerPlugin` interface (verify/parse/mapToTask); GitHub HMAC-SHA256; Perforce X-Xci-Token; Dead Letter Queue with manual retry; idempotency via delivery ID.
- **New: Web dashboard SPA** — React 19 + Vite + Tailwind 4 + shadcn/ui + TanStack Query + Zustand; agents, tasks, run history, live log viewer, org settings, plugin settings, DLQ; RoleGate disabled-not-hidden invariant; build-status badge endpoint.
- **New: Docker image + publishing** — multi-stage `node:22-slim` image; non-root UID 10001; migrations at boot; `@fastify/static` SPA serving; `docker-compose.yml` dev stack; CI smoke test (13 steps) before tag; Trivy scan; ghcr.io publish.
- **Preserved: xci CLI v1 backward compat** — all v1.0 behavior kept. `--agent` is opt-in; default CLI invocation is observably identical to v1 (BC-01..04).

See the [release runbook](./.github/RUNBOOK-RELEASE.md) for the operator checklist before cutting the first release.

### v1.0 — Local CLI (2026-04-15)

**Package:** `xci@1.0.0`

- Initial release of the `xci` CLI: 4-layer YAML config loader (machine → project → secrets → local), command resolver with alias composition and cycle detection, cross-platform executor (single / sequential / parallel) with AbortController cancellation, `xci init` scaffold, `--list` / `--dry-run` / `--verbose` / pass-through flags, `${VAR}` placeholder resolution with per-layer provenance, secrets redaction.
- Published on npm as `xci`.
- CI matrix: Ubuntu / Windows / macOS × Node 20 / 22.
