    **Create `src/commands/tokenize.ts`:**

    Export a single function `tokenize(input: string, aliasName: string): readonly string[]`.

    Implementation per D-03:
    - Iterate character by character through `input`
    - Track `inQuotes` boolean (toggled by `"` character)
    - On whitespace outside quotes: push accumulated token (if non-empty) and reset
    - On `"` outside quotes: set `inQuotes = true`, do NOT include the quote in the token
    - On `"` inside quotes: set `inQuotes = false`
    - On any other char: append to current token
    - After loop: if `inQuotes` is still true, throw `new CommandSchemaError(aliasName, 'unclosed double quote in command string')`
    - If final accumulated token is non-empty, push it
    - Return the token array
    - Import `CommandSchemaError` from `'../errors.js'`

    **Create `src/commands/normalize.ts`:**

    Export a function `normalizeCommands(raw: Record<string, unknown>, filePath: string): CommandMap`.

    For each `[aliasName, value]` in `Object.entries(raw)`:
    - Call `normalizeAlias(aliasName, value, filePath)` to get a `CommandDef`
    - Add to a `Map<string, CommandDef>`

    Internal `normalizeAlias(aliasName: string, raw: unknown, filePath: string): CommandDef`:

    1. If `typeof raw === 'string'`: return `{ kind: 'single', cmd: tokenize(raw, aliasName) }` (D-01 string shorthand)
    2. If `Array.isArray(raw)`: validate every element is a string, throw `CommandSchemaError(aliasName, 'array form must contain only strings')` if not. Return `{ kind: 'single', cmd: raw as readonly string[] }` (CMD-02 array form)
    3. If `typeof raw === 'object' && raw !== null`: call `normalizeObject(aliasName, raw as Record<string, unknown>, filePath)`
    4. Otherwise: throw `CommandSchemaError(aliasName, 'must be a string, array, or object')`

    Internal `normalizeObject(aliasName: string, obj: Record<string, unknown>, filePath: string): CommandDef`:

    - Check for `steps` key: if present, validate it is an array of strings, return `{ kind: 'sequential', steps: [...], description: obj.description as string | undefined }`
    - Check for `parallel` key: if present, validate it is an array of strings, return `{ kind: 'parallel', group: [...], description: obj.description as string | undefined }`
    - Otherwise treat as single command:
      - Extract `cmd` (may be string or array or undefined per D-14)
      - Extract `description` (optional string)
      - Extract platform overrides: check for `linux`, `windows`, `macos` keys. Each must be an object with a `cmd` key that is a string or string array. Call `tokenize()` on string platform cmds. Build `PlatformOverrides` object.
      - If `cmd` is undefined AND no platform overrides exist: throw `CommandSchemaError(aliasName, 'must have cmd, steps, or parallel')`
      - If `cmd` is string: tokenize it. If `cmd` is array: validate all strings.
      - If `cmd` is undefined but platform overrides exist: use empty array `[]` as default cmd (D-14)
      - Return `{ kind: 'single', cmd: [...], description, platforms }`

    Import `tokenize` from `'./tokenize.js'` and `CommandSchemaError` from `'../errors.js'`.
    Import types `CommandDef`, `CommandMap`, `PlatformOverrides` from `'../types.js'`.

    **Create `src/commands/validate.ts`:**

    Export a function `validateGraph(commands: CommandMap): void`.

    Implementation -- DFS with three-color marking per D-11:

    ```
    type Color = 'white' | 'gray' | 'black';

    function getAliasRefs(def: CommandDef): readonly string[] {
      // For 'sequential': return def.steps
      // For 'parallel': return def.group
      // For 'single': return [] (single commands have no alias refs)
    }
    ```

    Main algorithm:
    - Create `color: Map<string, Color>` initialized to 'white' for all keys
    - Create `path: string[]` stack
    - For each alias in `commands.keys()`:
      - If color is 'white', call `dfs(alias)`
    - `dfs(alias)`:
      - Set color to 'gray', push to path
      - Get the CommandDef from commands map
      - For each ref in `getAliasRefs(def)`:
        - If ref not in commands map: throw `new UnknownAliasError(ref)`
        - If color of ref is 'gray': extract cycle from path starting at `path.indexOf(ref)`, append ref to close loop, throw `new CircularAliasError(cyclePath)`
        - If color of ref is 'white': call `dfs(ref)`
      - Pop from path, set color to 'black'

    Also add depth cap enforcement (D-10):
    - Accept optional `depth` parameter in dfs, default 0
    - If `depth > 10`: throw `new CommandSchemaError(path[0], 'alias nesting exceeds maximum depth of 10: ' + path.join(' -> '))`
    - Pass `depth + 1` to recursive calls

    Note on D-09 (lookup-based alias detection): The step/group entries in `CommandDef` are `CommandRef` strings at this point. During validation, every entry in `steps`/`group` is checked against the CommandMap. If it matches a key, it IS an alias reference (and must be validated). If it does NOT match, it is an inline command string. The normalize step already stored it as a `CommandRef` either way -- the distinction matters only at resolution time (Plan 02). For validation purposes, only refs that match known aliases are followed in the DFS graph.

    Updated `getAliasRefs` to filter: return only entries from steps/group that exist as keys in the commands map. Unknown entries that are NOT alias names (i.e., inline commands) are not followed as edges. However, to satisfy CMD-09 (error on unknown alias ref), we need a heuristic: if a step/group entry looks like a simple word (no spaces, no slashes) AND is not a known alias, it could be an inline single-word command like "npm". Per D-09, the lookup is exact match only. So: entries matching a CommandMap key are alias refs; entries NOT matching are inline commands. CMD-09 says "error if a referenced alias does not exist" -- but the question is which entries ARE alias references. Per D-09: only those that match a key. Therefore `getAliasRefs` returns ONLY entries that are keys in the CommandMap. Entries that don't match are inline commands and are NOT validated as alias refs here.

    Wait -- re-reading CMD-09: "Il commands loader emette errore se un alias referenziato (in composizione) non esiste." This means: if a step/group entry is intended as an alias reference but the alias doesn't exist, error. Per D-09, the detection is lookup-based: if it matches a key, it's an alias ref. If it doesn't match, it's treated as an inline command. So there is NO case where CMD-09 fires for a non-existent alias -- because by definition, if it doesn't exist in the map, it's treated as inline. CMD-09 is satisfied by the DFS visiting only known aliases; unknown strings in steps/group are inline commands.

    **Replace `src/commands/index.ts` stub with real implementation:**

    ```typescript
    import { readFileSync } from 'node:fs';
    import { join } from 'node:path';
    import { parse, YAMLParseError as YamlLibError } from 'yaml';
    import { YamlParseError } from '../errors.js';
    import type { CommandMap, CommandsLoader } from '../types.js';
    import { normalizeCommands } from './normalize.js';
    import { validateGraph } from './validate.js';

    function readCommandsYaml(cwd: string): Record<string, unknown> | null {
      const filePath = join(cwd, '.loci', 'commands.yml');
      let raw: string;
      try {
        raw = readFileSync(filePath, 'utf8');
      } catch (err: unknown) {
        if (typeof err === 'object' && err !== null && (err as NodeJS.ErrnoException).code === 'ENOENT') return null;
        throw err;
      }
      let parsed: unknown;
      try {
        parsed = parse(raw);
      } catch (err: unknown) {
        if (err instanceof YamlLibError) {
          throw new YamlParseError(filePath, err.linePos?.[0]?.line, err);
        }
        throw err;
      }
      if (parsed === null || parsed === undefined) return null;
      if (typeof parsed !== 'object' || Array.isArray(parsed)) {
        throw new YamlParseError(filePath, undefined, new Error('Root document must be a YAML mapping'));
      }
      return parsed as Record<string, unknown>;
    }

    export const commandsLoader: CommandsLoader = {
      async load(cwd: string): Promise<CommandMap> {
        const raw = readCommandsYaml(cwd);
        if (raw === null) return new Map();
        const filePath = join(cwd, '.loci', 'commands.yml');
        const commands = normalizeCommands(raw, filePath);
        validateGraph(commands);
        return commands;
      },
    };
    ```
