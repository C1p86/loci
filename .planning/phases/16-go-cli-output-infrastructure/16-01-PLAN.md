Create `go-xci/internal/output/output.go` with `package output`. Imports allowed: `fmt`, `os`, `sort`, `regexp`, `github.com/fatih/color`, `github.com/mattn/go-isatty`, and `github.com/andrearuggeri/xci/internal/commands`. Do NOT import `internal/config`, `internal/resolver`, or `internal/executor` (D-01, Pitfall 5 — import cycle).

Implement exactly these exported symbols:

1. `func ShouldUseColor() bool` — priority per D-05:
   - If `NO_COLOR` env var is present (`_, ok := os.LookupEnv("NO_COLOR"); ok`), return false.
   - Else if `FORCE_COLOR` env var is present, return true.
   - Else return `isatty.IsTerminal(os.Stderr.Fd()) || isatty.IsCygwinTerminal(os.Stderr.Fd())`. (Check stderr, NOT stdout — Pitfall 1. go-isatty is already available via the fatih/color transitive dep added in Task 1.)

2. `func InitColor()` — sets the global `color.NoColor = !ShouldUseColor()`. Call once at startup (wired in Plan 02). This is the single control knob for all `color.New(...)` instances.

3. Package-level color objects (reused, not re-allocated per call):
   ```go
   var (
       boldCyan   = color.New(color.FgCyan, color.Bold)
       greenCheck = color.New(color.FgGreen)
       redCross   = color.New(color.FgRed)
       dimYellow  = color.New(color.FgYellow)
   )
   ```

4. `func PrintRunHeader(alias string, def commands.CommandDef, mergedValues map[string]string, secretKeys map[string]bool)` (D-02, D-03, D-04):
   - Print title to stderr: `boldCyan.Fprintf(os.Stderr, "▶ running: %s\n", alias)`.
   - Call `collectReferencedPlaceholders(def)` to get the referenced set.
   - Build a sorted slice of keys from `mergedValues` where `referenced[k]` is true (use `sort.Strings`).
   - If at least one referenced key exists, print `variables:` on its own line to stderr, then for each key print `  %s = %s` where the value is `**********` if `secretKeys[k]` is true (D-04), otherwise `mergedValues[k]`. All to `os.Stderr`.

5. `func PrintStepHeader(label string, stepNum, totalSteps int)` (D-07):
   - `boldCyan.Fprintf(os.Stderr, "▶ %s [%d/%d]\n", label, stepNum, totalSteps)`.

6. `func PrintStepCwd(cwd string)` (resolves Research Open Question 2 — avoids regression of the existing cwd display; gives Phase 17 a hook):
   - If `cwd == ""`, return without printing.
   - Else `dimYellow.Fprintf(os.Stderr, "  cwd: %s\n", cwd)`.

7. `type ParallelResult struct { Alias string; ExitCode int }` and `func PrintParallelSummary(results []ParallelResult)` (D-08):
   - Print a blank line to stderr first: `fmt.Fprintln(os.Stderr)`.
   - For each result: if `ExitCode == 0`, `greenCheck.Fprintf(os.Stderr, "  ✓ %s (exit 0)\n", r.Alias)`, else `redCross.Fprintf(os.Stderr, "  ✗ %s (exit %d)\n", r.Alias, r.ExitCode)`.

8. Unexported `collectReferencedPlaceholders(def commands.CommandDef) map[string]bool` (D-03, ports TypeScript `collectReferencedPlaceholders`):
   - Package-level `var placeholderRE = regexp.MustCompile(\`\$\{([^}]+)\}\`)`.
   - A local `scan(s string)` closure that runs `placeholderRE.FindAllStringSubmatch(s, -1)` and adds each capture group `m[1]` to the output set.
   - `switch def.Kind`: `KindSingle` → scan each string in `def.Cmd`; `KindSequential` → scan each string in `def.Steps`; `KindParallel` → scan each string in `def.Group`. Return the set.

ALL writes go to `os.Stderr` (D-09). Never write to `os.Stdout` from this package.

Then create `go-xci/internal/output/output_test.go` (`package output`) implementing the tests described in the behavior block. Since `collectReferencedPlaceholders` is unexported, the test file must be in `package output` (white-box) to access it. Use `t.Setenv` for env-var tests so they auto-restore. For collectReferencedPlaceholders tests, construct `commands.CommandDef` literals directly.
