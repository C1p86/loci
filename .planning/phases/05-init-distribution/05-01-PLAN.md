Create `src/init/index.ts` with:

1. A `SummaryItem` type: `{ path: string; action: 'created' | 'skipped' | 'updated' }`

2. A `writeIfAbsent(filePath: string, content: string, results: SummaryItem[]): void` helper that:
   - Uses `existsSync(filePath)` to check
   - If exists: push `{ path: relative, action: 'skipped' }`
   - If not: `writeFileSync(filePath, content, 'utf8')` then push `{ path: relative, action: 'created' }`

3. A `ensureGitignore(projectDir: string, results: SummaryItem[]): void` function per D-05/D-06/D-07:
   - `GITIGNORE_ENTRIES = ['.loci/secrets.yml', '.loci/local.yml']`
   - If `.gitignore` does not exist: create it with `# loci\n.loci/secrets.yml\n.loci/local.yml\n`; push action 'created'
   - If `.gitignore` exists: read it, split by `\n`, `.map(l => l.trim())` (handles CRLF per Pitfall 2), check if each entry is already present, append missing with `\n# loci\n{missing entries}\n`, push action 'updated' or 'skipped'

4. A `printInitSummary(results: SummaryItem[]): void` function that uses `process.stdout.write()`:
   ```
   loci init

     created  .loci/config.yml
     created  .loci/commands.yml
     skipped  .gitignore

   Run `loci hello` to test your setup.
   ```
   Format: `  ${action.padEnd(8)} ${path}\n`

5. A `runInit(cwd: string): void` function (synchronous, per RESEARCH anti-pattern guidance):
   - `const lociDir = join(cwd, '.loci')`
   - `mkdirSync(lociDir, { recursive: true })` (idempotent per Pitfall 3)
   - Call `writeIfAbsent` for each of the 4 files:
     - `join(lociDir, 'config.yml')` with `CONFIG_YML`
     - `join(lociDir, 'commands.yml')` with `COMMANDS_YML`
     - `join(lociDir, 'secrets.yml.example')` with `SECRETS_EXAMPLE_YML`
     - `join(lociDir, 'local.yml.example')` with `LOCAL_EXAMPLE_YML`
   - Call `ensureGitignore(cwd, results)`
   - Call `printInitSummary(results)`

6. A `registerInitCommand(program: Command): void` function:
   - `program.command('init').description('Scaffold a .loci/ directory in the current project').action(() => { runInit(process.cwd()); })`
   - The action is synchronous (no async needed -- all fs ops are sync)

Export: `runInit`, `registerInitCommand`

All imports use `node:fs` and `node:path`. Use `.js` suffix on local imports (e.g., `'./templates.js'`) per project convention (verbatimModuleSyntax).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem writes | init writes files to user's project directory |
| .gitignore modification | appending to an existing user file |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | .gitignore append | mitigate | Only append known-safe entries (`.loci/secrets.yml`, `.loci/local.yml`); read file content and check for existing entries before appending; never overwrite existing .gitignore content |
| T-05-02 | Information Disclosure | scaffold templates | accept | Templates contain only example placeholder text with no real secrets; `.example` suffix files are safe to commit |
| T-05-03 | Denial of Service | mkdirSync | accept | init creates 1 directory + 4 small files; negligible disk impact |
| T-05-04 | Tampering | writeIfAbsent | mitigate | existsSync guard ensures existing files are never overwritten (idempotency); no user data is modified |
</threat_model>
