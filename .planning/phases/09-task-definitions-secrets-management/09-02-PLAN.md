<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| XCI_MASTER_KEY env var -> fastify.mek Buffer | Single parse point (app.ts boot). A leak here compromises all org DEKs. |
| Plaintext secret arg -> AES-GCM cipher | In-memory only within encryptSecret; must never be logged. |
| Auth-tag verification -> plaintext return | decryptSecret is the sole producer of plaintext from stored ciphertext; tag mismatch must never silently degrade to "return corrupted plaintext". |
| DEK plaintext -> wrap/unwrap functions | DEK lives in memory as a 32-byte Buffer for microseconds; must never be serialised, logged, or returned in an error. |
| Pino log pipeline -> redacted request bodies | req.body.value / req.body.newMekBase64 must be scrubbed BEFORE the log line is emitted. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-02-01 | Information Disclosure | XCI_MASTER_KEY in logs | mitigate | @fastify/env + pattern validation fail at boot; no log statement handles `app.config.XCI_MASTER_KEY`; redact.paths includes *.mek (catches any accidental field spread) |
| T-09-02-02 | Information Disclosure | Plaintext secret value in log | mitigate | redact.paths extended with `req.body.value` (POST/PATCH /secrets) + `*.ciphertext` + `*.dek`; SEC-04 architectural invariant enforced by Plan 09-05 CI grep |
| T-09-02-03 | Tampering | AES-GCM IV reuse -> catastrophic key-stream leak | mitigate | encryptSecret and wrapDek both call randomBytes(12) on EVERY invocation; unit test asserts notDeepEqual(encrypt(k,'x').iv, encrypt(k,'x').iv); SEC-02 |
| T-09-02-04 | Tampering | Auth-tag skip -> forged ciphertext accepted | mitigate | decryptSecret/unwrapDek call setAuthTag BEFORE update/final (Pitfall 1); on any crypto failure throw SecretDecryptError — never fall through |
