<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Untrusted HTTP request body -> AJV schema validation | JSON body validated before reaching handler. Bounds (name maxLength 255, yamlDefinition maxLength 1MB) defend against resource exhaustion. |
| Request session -> org membership guard | requireOwnerOrMemberAndOrgMatch enforces that req.org.id matches the :orgId param AND the role is not Viewer. Session is validated by Phase 7 auth plugin. |
| CSRF token cookie+header pair -> mutating operation | @fastify/csrf-protection double-submit — missing or mismatched token -> 403. Applied to POST/PATCH/DELETE. |
| YAML text from request -> xci/dsl facade | Untrusted YAML parsed via `yaml` lib (YAML 1.2 semantics, bomb-resistant). No eval. Parser errors contained in lenient { errors[] } return. |
| TaskValidationError response body -> API consumer | Contains line/column/message/suggestion — pre-sanitised by the dsl parser (never the raw YAML body). Safe for UI display. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-04-01 | Tampering | CSRF on POST/PATCH/DELETE /api/orgs/:orgId/tasks | mitigate | onRequest [fastify.csrfProtection] on all 3 mutating routes (Phase 7 D-34 opt-in) |

| T-09-04-05 | Denial of Service | Deeply nested alias graph causing stack exhaustion in cycle detection | mitigate | v1 validateGraph is iterative 3-color DFS with depth cap (Phase 3 D-10 = 10); no recursion explosion risk |
