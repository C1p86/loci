### Task 2: 3 Org-Scoped Repos + forOrg Wiring + Isolation Tests

**`makeWebhookTokensRepo(db, orgId, mek: Buffer)`** — methods: `list()`, `getById(id)`, `create({pluginName, pluginSecret?, createdByUserId?})`, `resolvePluginSecret(id)`, `revoke(id)`, `delete(id)`.

- `create`: generates random plaintext via `generateToken()`, stores `sha256` hash only, encrypts `pluginSecret` via `getOrCreateOrgDek` + `encryptSecret` with AAD=`${orgId}:webhook:${tokenId}`. Returns `{id, plaintext, endpointPath}`. Plaintext never stored.
- `resolvePluginSecret`: fetches DEK, calls `decryptSecret`, returns Buffer. Returns `null` for Perforce (no HMAC). Returns `undefined` if revoked or wrong org.
- All queries scoped by `eq(webhookTokens.orgId, orgId)`.

**`makeWebhookDeliveriesRepo(db, orgId)`** — methods: `recordDelivery({pluginName, deliveryId})`, `isDuplicate(pluginName, deliveryId)`.

- `recordDelivery`: INSERT + `.onConflictDoNothing({target: [pluginName, deliveryId]}).returning()`. Returns `{inserted: true, id}` or `{inserted: false, id: null}` on duplicate.

**`makeDlqEntriesRepo(db, orgId)`** — methods: `create(params)`, `getById(id)`, `list(opts)`, `markRetried(id, result)`.

- `list`: cursor-based pagination using raw `sql` template for the OR condition, ordered `DESC receivedAt, id`. Limit clamped to max 200.

**forOrg wiring:** Added `webhookTokens: makeWebhookTokensRepo(db, orgId, mek)`, `webhookDeliveries: makeWebhookDeliveriesRepo(db, orgId)`, `dlqEntries: makeDlqEntriesRepo(db, orgId)` to `makeForOrg` return.

**3 isolation test files** (D-04 auto-discovery):
- `webhook-tokens.isolation.test.ts`: 6 tests — list/getById cross-tenant isolation, hash-only storage check, resolvePluginSecret across orgs, revoke cross-tenant no-op, revoked token returns undefined.
- `webhook-deliveries.isolation.test.ts`: 5 tests — insert success, same-org duplicate dedup, cross-plugin non-duplicate, isDuplicate behavior, isDuplicate org-scoping.
- `dlq-entries.isolation.test.ts`: 6 tests — list/getById isolation, markRetried cross-tenant no-op, markRetried within org, cursor pagination, limit clamp.

### Task 3: adminRepo Cross-Org Helpers + Integration Tests

**`findWebhookTokenByPlaintext(tokenPlaintext: string)`** — D-06 cross-org lookup:
```ts
const tokenHash = hashToken(tokenPlaintext);
SELECT orgId, id AS tokenId, pluginName
FROM webhook_tokens
WHERE token_hash = $hash AND revoked_at IS NULL
LIMIT 1
```
Returns `{orgId, tokenId, pluginName}` or `undefined`. Plaintext never stored in returned object (ATOK-06).

**`cleanupDeliveries(before: Date)`** — D-22 retention:
```ts
DELETE FROM webhook_deliveries WHERE received_at < $before
RETURNING id
```
Returns `{rowsDeleted: result.length}`.

**`admin.webhooks.integration.test.ts`** (8 tests): hit, miss (unknown), miss (revoked), ATOK-06 structural check, cleanup with 2 old + 2 recent rows (deletes 2), no-op when nothing matches.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Drizzle `or()` return type incompatible with `conditions` array**
- **Found during:** Task 2 (dlq-entries.ts cursor pagination)
- **Issue:** `or()` from drizzle-orm returns `SQL<unknown> | undefined`, which TypeScript rejects when pushed to `SQL<unknown>[]`
- **Fix:** Replaced `or(lt(...), and(...))` with a raw `sql` template that encodes the OR condition directly: `sql\`(receivedAt < cursor OR (receivedAt = cursor AND id < cursor.id))\``
- **Files modified:** `packages/server/src/repos/dlq-entries.ts`
- **Commit:** `2b6622e`
