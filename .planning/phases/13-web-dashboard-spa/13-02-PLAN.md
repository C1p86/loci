<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /api | Session cookie carries auth; CSRF header must be sent on mutations |
| SPA bundle → user's browser | Must not include server-side secrets; env vars prefixed VITE_ only |
| packages/web/src → other workspace packages | Biome fence forbids xci/server/agent/dsl imports |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-02-01 | I (Info Disclosure) | packages/web/src/** | mitigate | Biome noRestrictedImports blocks imports of `xci`, `xci/dsl`, `xci/agent`, `@xci/server` from web/src. Prevents Node types (FastifyRequest, Buffer, MEK) leaking into browser bundle. Per D-05 / D-41. |
| T-13-02-02 | T (Tampering) | Client-side auth state | mitigate | authStore is a derivative of /api/auth/me — any mutation to authStore on client is UX only; server-side session cookie is authoritative. RoleGate disabling is UX only; server re-checks role on every mutation (Phase 7 AUTH-10). |
| T-13-02-03 | S (Spoofing) | CSRF on mutations | mitigate | api.ts apiPost/Patch/Delete automatically attaches `X-CSRF-Token` header from cookie (double-submit pattern per Phase 7 D-34). apiGet does not send CSRF. |
| T-13-02-04 | I (Info Disclosure) | Browser localStorage | accept | Only uiStore (non-sensitive UX prefs) persisted to localStorage. authStore + wsStore are in-memory only. Never persist user email, plan, or session indicators. |
| T-13-02-05 | R (Repudiation) | Console logs in prod build | mitigate | Vite prod build strips `console.debug/info`; errors bubble to error boundary which reports to... nowhere (v2.0 has no error telemetry). Leave console.error in place for browser devtools. |
| T-13-02-06 | E (Elevation) | XSS via user-supplied strings | mitigate | React auto-escapes. NEVER use dangerouslySetInnerHTML in this plan. Lint rule will block it (Biome noReactDangerouslySetInnerHTML). |
| T-13-02-07 | D (DoS) | Dependency supply chain | mitigate | Pinned versions (no ^ / ~); pnpm lockfile committed; dev deps only via pnpm workspace root; no postinstall scripts allowed (pnpm --ignore-scripts enforced on CI). |

## Security hardening mandatory in this plan
- [x] `credentials: 'include'` on every fetch (session cookie required)
- [x] `X-CSRF-Token` header on every non-GET fetch (double-submit pattern)
- [x] No `dangerouslySetInnerHTML` anywhere in src/ (Biome rule + grep gate)
- [x] No cross-package imports from web/src/ (Biome noRestrictedImports)
- [x] Content-Security-Policy note in index.html for future-proofing (meta tag placeholder — actual CSP set by server in Phase 14 Docker)
</threat_model>

    import { readCookie } from './utils.js';

    export class ApiError extends Error {
      constructor(public code: string, public status: number, message: string, public details?: unknown) {
        super(message);
      }
    }

    async function request<T>(method: string, url: string, body?: unknown): Promise<T> {
      const headers: Record<string, string> = { 'Accept': 'application/json' };
      if (body !== undefined) headers['Content-Type'] = 'application/json';
      if (method !== 'GET' && method !== 'HEAD') {
        const csrf = readCookie('xci_csrf');
        if (csrf) headers['X-CSRF-Token'] = csrf;
      }
      let res: Response;
      try {
        res = await fetch(url, {
          method,
          headers,
          credentials: 'include',
          body: body === undefined ? undefined : JSON.stringify(body),
        });
      } catch (err) {
        throw new ApiError('NETWORK_ERROR', 0, (err as Error).message);
      }
      const text = await res.text();
      const parsed = text ? JSON.parse(text) : null;
      if (!res.ok) {
        throw new ApiError(parsed?.error ?? 'HTTP_ERROR', res.status, parsed?.message ?? res.statusText, parsed);
      }
      return parsed as T;
    }

    export const apiGet    = <T>(url: string)                 => request<T>('GET', url);
    export const apiPost   = <T>(url: string, body?: unknown) => request<T>('POST', url, body);
    export const apiPatch  = <T>(url: string, body?: unknown) => request<T>('PATCH', url, body);
    export const apiDelete = <T>(url: string)                 => request<T>('DELETE', url);

    import type { ReactNode } from 'react';
    import { useAuthStore } from '../stores/authStore.js';
    import { DisabledWithTooltip } from './DisabledWithTooltip.js';
    import type { Role } from '../lib/types.js';

    const ROLE_RANK: Record<Role, number> = { viewer: 0, member: 1, owner: 2 };

    interface RoleGateProps {
      role: Role;                       // minimum role required
      children: ReactNode;              // the mutation control
      fallback?: ReactNode;             // optional custom fallback
      tooltip?: string;                 // default: "Viewers/Members cannot ..."
    }

    export function RoleGate({ role, children, fallback, tooltip }: RoleGateProps) {
      const currentRole = useAuthStore((s) => s.org?.role ?? 'viewer');
      const allowed = ROLE_RANK[currentRole] >= ROLE_RANK[role];
      if (allowed) return <>{children}</>;
      if (fallback) return <>{fallback}</>;
      const reason = tooltip ?? `Your role (${currentRole}) cannot perform this action. Requires ${role} or higher.`;
      return <DisabledWithTooltip reason={reason}>{children}</DisabledWithTooltip>;
    }
