## Architecture Highlights

### Scoped Repository Wrapper (D-01)

The only way to query org-scoped tables is through `forOrg(orgId)`. Importing a sibling repo file directly from a route handler is blocked by Biome's `noRestrictedImports` rule. Example:

```ts
import { makeRepos } from '@xci/server/repos';

const repos = makeRepos(fastify.db);
const { users } = repos.forOrg(request.org.id);
const user = await users.findById(userId); // automatically includes `org_id = request.org.id` filter
```

Cross-org operations (signup, cross-org lookups) use the deliberately friction-ful `repos.admin` namespace.

### D-04 Auto-Discovery Isolation Test

`src/repos/__tests__/isolation-coverage.isolation.test.ts` walks `src/repos/` at test time and fails CI if any public `makeXxxRepo()` factory lacks a corresponding `<name>.isolation.test.ts` or the test doesn't reference the factory by name. Adding a new repo function without a test → CI red.

### Fastify Plugin Order (D-06, locked)

```
@fastify/env → db plugin → @fastify/helmet → @fastify/cookie → @fastify/csrf-protection → @fastify/rate-limit → auth plugin → error-handler → routes
```

Each plugin declares `dependencies` so Fastify enforces the order at register time. `@fastify/autoload` is deliberately NOT used.
