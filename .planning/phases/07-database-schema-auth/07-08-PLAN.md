```yaml
  integration-tests:
    # Linux-only (D-23) — testcontainers needs Docker, preinstalled on ubuntu-latest.
    # Gates on build-test-lint matrix passing first (no point running integration if unit/build is broken).
    needs: [build-test-lint]
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup pnpm
        uses: pnpm/action-setup@v4

      - name: Setup Node.js 22
        uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: 'pnpm'

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Build @xci/server
        run: pnpm --filter @xci/server build

      - name: Run integration suite (testcontainers Postgres 16-alpine)
        run: pnpm --filter @xci/server test:integration
        # First run: ~30-45s container pull + boot + full suite
        # Subsequent runs: ~15-20s (Docker layer cache on ubuntu-latest is ephemeral between jobs,
        # but postgres:16-alpine image is usually cached at the runner host level)

      - name: Upload vitest results on failure
        if: failure()
        uses: actions/upload-artifact@v4
        with:
          name: vitest-integration-results
          path: packages/server/.vitest-output/
          if-no-files-found: ignore
          retention-days: 7
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → repo code | Workflow runs against the PR's commit; testcontainers spawns Postgres in the runner's Docker sandbox |
| Branch protection policy → merge button | Required-check list is the enforcement point; without adding integration-tests, AUTH-10 isn't gated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-08-01 | Tampering | PR bypasses test:integration because the matrix `test` script no longer runs it | mitigate | Explicit `integration-tests` job in CI runs `pnpm --filter @xci/server test:integration` as an independent required check. Task 1 verify step confirms `grep -q "pnpm --filter @xci/server test:integration" .github/workflows/ci.yml`. |
| T-07-08-02 | Elevation of Privilege | Attacker opens a PR that disables the integration-tests job | mitigate | Branch protection rule requires `integration-tests` check to be green (post-checkpoint). Modifying ci.yml to remove the job would trigger the check against the modified workflow and either (a) fail because job missing → required check never runs, blocking merge; (b) GitHub's "Require status checks" policy requires a successful run of the named check → never runs → never mergeable. |
| T-07-08-03 | Information disclosure | Test artifacts (vitest-integration-results) expose sensitive data on failure | accept | vitest output contains test names + stack traces + DB queries. Pino redaction (Plan 05) applies to server logs, not vitest stdout. Integration tests use stub email transport and synthetic data — no real secrets. Artifact retention 7 days. |
| T-07-08-04 | DoS | testcontainers pull postgres:16-alpine exceeds CI time budget | accept | First run ~30-45s (pull + boot + migrations + ~50 tests). GitHub Actions default job timeout 6h. Well within budget. |
| T-07-08-05 | Tampering | Dependency version drift between dev and CI | mitigate | `pnpm install --frozen-lockfile` in the new job (identical to existing jobs) ensures lockfile integrity. Exact versions pinned in Plan 01. |

Severity: all low/medium. Checkpoint is the critical action — without it, the gate isn't enforced.
</threat_model>
