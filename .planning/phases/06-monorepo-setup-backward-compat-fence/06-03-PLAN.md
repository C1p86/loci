  "ignore": []

```

Field explanations (for executor context — do not edit the file to add comments, JSON does not support them):
- `"fixed": [["xci", "@xci/server", "@xci/web"]]` — the three packages always release at the same version (D-11). This is a 2D array: each sub-array is one "fixed group". We have one group with three members.
- `"access": "public"` — applies to scoped packages (`@xci/server`, `@xci/web`). Unscoped `xci` ignores this field (defaults to public anyway). Belt-and-suspenders because the release script also passes `--access=public`.
- `"baseBranch": "main"` — Changesets uses this to determine which packages changed since the last release when running `changeset status`.
- `"changelog": "@changesets/cli/changelog"` — the default generator; sufficient for Phase 6.
- `"commit": false` — Changesets does NOT auto-commit; the developer commits the .changeset/*.md file manually.
- `"ignore": []` — no packages to skip. We do NOT list `@xci/server` / `@xci/web` here because their `"private": true` flag already prevents them from publishing (D-12 amendment + Pitfall 7 option 1).

`.changeset/README.md` with this content (lightweight stub — the canonical `pnpm changeset init` generates something similar; we write it by hand to avoid requiring a pre-install run):

```markdown
# Changesets

This directory is managed by [Changesets](https://github.com/changesets/changesets).

## Recording a change

When you make a change to any package that should trigger a release:

```bash
pnpm changeset
```

Follow the prompts. The tool writes a markdown file in this directory (e.g. `.changeset/fluffy-llamas-sing.md`). Commit it along with your code change.

## Releasing

1. On push to `main`, the `Release` GitHub Actions workflow runs `changesets/action@v1`.
2. The action aggregates all pending changesets and opens a "Version PR" that bumps package versions and updates CHANGELOG entries.
3. When the Version PR is merged, the action runs `pnpm -r publish --access=public` to push all three packages (`xci`, `@xci/server`, `@xci/web`) to npm.

## Fixed-versioning

`xci`, `@xci/server`, and `@xci/web` are locked together via `.changeset/config.json` → `"fixed": [["xci", "@xci/server", "@xci/web"]]`. A bump to any one bumps all three.

In Phase 6, `@xci/server` and `@xci/web` are `"private": true` — they are NOT published. Real implementations land in Phase 9+ (server) and Phase 13+ (web), at which point their `private` flag flips to `false` in the same commit that ships their code.
```
