### `packages/xci/tsup.config.ts` (config, MIGRATED + MODIFIED)

**Analog:** `/home/developer/projects/loci/tsup.config.ts` (31 lines)

**Current content (verbatim)** (lines 1-31):
```typescript
import { readFileSync } from 'node:fs';
import { defineConfig } from 'tsup';

const pkg = JSON.parse(readFileSync('./package.json', 'utf8')) as { version: string };

export default defineConfig({
  entry: ['src/cli.ts'],
  format: ['esm'],
  target: 'node20.5',
  outDir: 'dist',
  outExtension: () => ({ js: '.mjs' }),
  bundle: true,
  noExternal: [/.*/],
  clean: true,
  dts: false,
  sourcemap: false,
  minify: false,
  splitting: false,
  treeshake: true,
  banner: {
    // Shebang must be the literal first line for Unix exec() to recognize the interpreter.
    // The createRequire polyfill lets esbuild's `__require` shim resolve Node builtins like
    // `require('events')` when bundling CJS deps (commander) into an ESM output.
    js: "#!/usr/bin/env node\nimport { createRequire as __xci_createRequire } from 'node:module';\nconst require = __xci_createRequire(import.meta.url);",
  },
  define: {
    __XCI_VERSION__: JSON.stringify(pkg.version),
  },
  platform: 'node',
});
```

**Patterns to preserve (DO NOT change):**
- `entry: ['src/cli.ts']` — path is package-relative; stays the same after migration
- `format: ['esm']` — ESM only
- `target: 'node20.5'` — matches engines floor
- `outExtension: () => ({ js: '.mjs' })` — preserves `.mjs` suffix
- `bundle: true` (CLAUDE.md §Cold-Start Budget)
- `banner.js` — entire shebang + createRequire polyfill block stays BYTE-FOR-BYTE identical (Phase 1 P02 decision, preserved in STATE.md)
- `define: { __XCI_VERSION__: JSON.stringify(pkg.version) }` — injected at bundle time, read by `src/version.ts`
- `platform: 'node'`, `treeshake: true`, `clean: true`, `dts: false`, `sourcemap: false`, `minify: false`, `splitting: false` — all unchanged

**Patterns to change (per RESEARCH.md §Pitfall 1):**

Line 13 MUST change. Current `noExternal: [/.*/]` takes precedence over `external` — a naive addition of `external: ['ws', 'reconnecting-websocket']` will NOT work. Planner picks one of:

Option A (explicit regex exclusion, keeps "bundle everything" semantics):
```typescript
noExternal: [/^(?!ws$|reconnecting-websocket$).*/],
external: ['ws', 'reconnecting-websocket'],
```

Option B (drop `noExternal`, rely on tsup default + explicit external for documentation):
```typescript
// no noExternal line at all
external: ['ws', 'reconnecting-websocket'],
// tsup default bundles declared `dependencies` (commander/execa/yaml),
// externalizes everything else. Since ws/reconnecting-websocket are NOT
// in dependencies of packages/xci/package.json, they'd be externalized anyway;
// the explicit `external` line is belt-and-suspenders + documentation intent.
```

RESEARCH.md recommends Option B as cleaner. Executor must verify fresh `pnpm --filter xci build` yields dist/cli.mjs of expected size (≈126 KB baseline per STATE.md; CONTEXT specifics note current observed 775 KB is likely a stale dev build).
