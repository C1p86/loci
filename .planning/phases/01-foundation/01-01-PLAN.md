## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| npm registry → local install | Transitive dep code runs on `npm install` (postinstall scripts) |
| Contributor machine → published tarball | `files:` field controls what ships |
| Build toolchain → bundle output | tsup define / banner control what the final .mjs contains |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | Supply chain (npm transitive deps) | mitigate | Runtime deps pinned EXACTLY in package.json (commander=14.0.3, execa=9.6.1, yaml=2.8.3 — no caret). `package-lock.json` committed so CI `npm ci` fails on drift. Minimal direct dep surface: only 3 runtime + 5 dev deps. |
| T-01-02 | Tampering | package.json postinstall scripts | mitigate | package.json has NO `postinstall` / `preinstall` / `prepare` scripts — only `prepublishOnly` (runs on maintainer machine only, not contributors). Verified by task acceptance: grep the manifest for script names. |
| T-01-03 | Information Disclosure | Published tarball contents | mitigate | `"files": ["dist", "README.md"]` excludes src/, tests/, .github/, .planning/, node_modules/, configs. Phase 5 will verify with `npm pack --dry-run` before publishing. |
