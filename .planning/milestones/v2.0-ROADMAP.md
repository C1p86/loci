# Milestone v2.0: Remote CI — Agents + Web Dashboard

**Status:** ✅ SHIPPED 2026-04-19
**Phases:** 6–14 (9 phases)
**Total Plans:** 50 plans
**Requirements:** 99/99

## Overview

v2.0 transforms `xci` from a local CLI tool into a distributed CI system: agents that register to a web SaaS server, tasks defined via UI and dispatched remotely to agents — preserving the v1 CLI without observable regressions.

**Scope pivots from v1.0 Out-of-Scope:**
- Remote execution (via agent WebSocket) — INVERTED ✓
- Automatic triggers (GitHub/Perforce webhook plugins) — INVERTED ✓
- Web UI dashboard (React SPA) — INVERTED ✓
- Secrets via vault/KMS (envelope encryption hybrid) — PARTIALLY INVERTED ✓

### Phase 8: Agent Registration & WebSocket Protocol

**Goal**: An agent can register with a one-time token, maintain a persistent WS connection with heartbeat/reconnect, and the server accurately tracks online/offline/draining/reconciled state
**Depends on**: Phase 7
**Requirements**: ATOK-01 through ATOK-06, AGENT-01 through AGENT-08
**Success Criteria**:
  1. `xci --agent <url> --token <reg-token>` registers agent, stores credential locally, goes online; subsequent restarts reconnect without re-registering
  2. Revoked credential rejected within one reconnect cycle (WS closes "revoked", agent exits non-zero)
  3. After disconnect, agent reconnects with exponential backoff; server reconciles in-flight task state
  4. Agent in drain mode receives no new dispatches; current tasks complete before agent stops
  5. SIGTERM sends `goodbye` frame, completes tasks, exits 0 with no orphaned runs

## Milestone Summary

**Timeline:** 2026-04-16 → 2026-04-19 (4 intensive days)
**Stats:** 564 files changed, 115,103 insertions, 26,364 deletions

**Key Accomplishments:**
1. Monorepo restructured with pnpm workspaces + Turborepo + Changesets; 3-layer ws-exclusion fence + v1 backward-compat active from day one
2. Full multi-tenant auth stack: Fastify + Drizzle + Postgres, signup/login/sessions/invites, two-org isolation test fixture enforces org_id filter on every repo function
3. Agent mode: TOFU registration, persistent WebSocket with 25s heartbeat, exponential backoff reconnect, full lifecycle (online/offline/drain/reconcile/shutdown)
4. Server-side task definitions using shared YAML DSL (xci/dsl); org-level envelope encryption (AES-256-GCM, MEK/DEK model), no plaintext secret ever returned via API
5. Label-match dispatch pipeline with 8-state TaskRun machine, per-org quota enforcement (Free: 5 agents / 5 concurrent), timeout/cancel/orphan recovery
6. Real-time log chunk streaming: Postgres persistence, RunBuffer in-memory fanout, ordered replay via sinceSeq, secret redaction pre-persist (server + agent-side)
7. GitHub + Perforce webhook plugins (HMAC/token verify, DLQ, idempotency via delivery_id, header scrubbing); Node-free Perforce trigger scripts
8. React 19 + Vite + Tailwind 4 SPA: agents list, task YAML editor (Monaco), live log viewer, run history, org settings, plugin settings, build-status badges
9. Multi-stage Docker image (<400MB, non-root UID), docker-compose dev stack, CI smoke test, npm publish pipeline (Changesets), release runbook

**Key Decisions:**
- Build tool for @xci/server is `tsc -b` (not tsup) — servers have no cold-start pressure
- ws/reconnecting-websocket kept external in tsup; agent.mjs is a separate bundle entry
- Agent token transmitted in WS frame body only, never in connection URL (proxy log safety)
- forOrg(orgId) is the sole entry point into org-scoped repos — Biome noRestrictedImports enforces the boundary structurally
- CSRF is per-route, NOT global; signup/login CSRF-exempt (no session yet)
- Admin dummy argon2 verify on unknown-email login equalizes timing (anti-enumeration)
- LogBatcher timer NOT reset on subsequent enqueues — original 200ms budget is the guarantee
- Bundle-size CI gate deferred (baseline 760KB vs 200KB; threshold needs revision in v2.1+)
- docker.yml tag-triggered (v*.*.*); Trivy scan (HIGH/CRITICAL block, ignore-unfixed) before push

**Technical Debt:**
- Bundle-size CI gate not wired (D-15 omitted per user decision in Phase 6; ws-fence still active via other layers)
- Session token not hashed at rest (D-12 deferred to v2.1)
- haveibeenpwned password check deferred (D-32)
- multi-instance scaling requires Redis pub/sub for agentRegistry (currently in-memory, single-process)
- 68 pre-existing Biome style errors in packages/xci/src/ (byte-identical to v1 tag — cosmetic only)

**Deferred to v2.1+:**
- Stripe integration and paid plans (FUT-01)
- KMS real integration — AWS KMS / GCP KMS / Vault (FUT-07)
- Multi-region / HA deploy (FUT-08)
- Matrix runs and artifact passing (FUT-02)
- SSO / OIDC / 2FA (FUT-04)
- Global log search (FUT-03)
