<objective>
Fix the `kind: xci` delegate so the delegated command's real output is both SHOWN on the outer terminal and SAVED to the outer project's `.xci/log/` file, exactly like a normal command step — while preserving the existing anti-hang behavior, nesting attenuation, exit-code propagation, and secret redaction.

Purpose: Today the delegate spawns with `stdio: 'inherit'` and gets no logFile, so the outer neither tees to the terminal nor logs; and the nested inner (quiet, attenuated) writes its real output only to its OWN logfile, so even `inherit` shows nothing. The fix tees piped output through the outer AND forwards an output-intent flag so the inner streams.

Output: A shared tee helper (`tee.ts`), an updated `runXciDelegate` with piped+exit-event-resolved spawn, updated call sites, extended unit + e2e tests, a README note, and a patch changeset.
</objective>
