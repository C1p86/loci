<objective>
Remove the `TERMINAL_STATES` guard in `LogViewer` so the WebSocket opens for terminal runs too.
The server already handles the replay + end-frame + graceful close path (confirmed in
`packages/server/src/routes/runs/logs-ws.ts`), so the frontend gate is the only thing
preventing terminal-state runs from ever seeing their log history.

Purpose: Users viewing a completed run currently see an empty log pane because the WS is
never opened. Server-side replay + end-frame was built specifically for this case — we
just need to stop blocking it on the client.

Output: A 2-line diff in `LogViewer.tsx` (plus optional removal of the now-unused
`TERMINAL_STATES` constant). No other files change.
</objective>
