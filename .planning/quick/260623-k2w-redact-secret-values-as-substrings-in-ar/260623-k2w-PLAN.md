<objective>
SECURITY FIX: redact secret values as SUBSTRINGS within argv tokens and cwd strings, not only as whole-token exact matches.

Today `redactArgv` does `secretValues.has(token) ? '***' : token`, so the common xci pattern `args: [token=${DEPLOY_TOKEN}]` — which interpolates to the single token `token=s3cr3t-abc123` — is NOT equal to the secret value `s3cr3t-abc123` and is NOT redacted. The secret leaks in cleartext through the delegation banner, printRunHeader (`▶ running:` / `xci →`), printDryRun, and printVerboseCommand (all route through redactArgv/redactCwd). This violates CLAUDE.md's hard rule: "Loci NON deve mai loggare i valori dei secrets."

Purpose: Close a confirmed cleartext secret leak in xci stderr output.
Output: A shared substring redactor used by both redactArgv and redactCwd, full test coverage, an e2e proof, an updated README, and a patch changeset.
</objective>
