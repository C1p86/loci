<objective>
Fix CTRL+C (SIGINT) handling so an intentional user abort does not trigger inappropriate completion UX.

When CTRL+C interrupts a running command, the child process exits with code 130 (POSIX 128 + signal 2 = SIGINT). The current code treats 130 like any other failure: it fires a "errore (exit 130)" toast and shows the "Show log? [y/N]" prompt. Both are wrong for a deliberate abort.

Purpose: Make abort behavior match user intent — silent on toast, no log prompt, plus a short stderr confirmation.
Output: Two gated call sites (`executor/index.ts`, `cli.ts`) plus an abort confirmation message.
</objective>
