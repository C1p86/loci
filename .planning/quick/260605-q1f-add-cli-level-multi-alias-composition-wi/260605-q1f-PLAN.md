Output: An extracted `runAlias()` core in cli.ts reused by both the existing single-alias action handler and a new multi-alias pre-parser in `main()`, plus e2e tests.

  <name>Task 2: Add multi-alias + pre-parser in main()</name>
  <files>packages/xci/src/cli.ts</files>
  <action>
Add a multi-alias pre-parser in `main(argv)` that runs AFTER config+commands are loaded (i.e. after `registerAliases(program, commands, config, projectRoot)` near line 1027) but BEFORE `program.parseAsync`. It only activates when `+` is present as a standalone argv token in the user-arg region.

Implementation:

1. Compute the user-arg region: `const userArgv = argv.slice(2);` (skip node + script path).

2. Detect a `+` separator that is a standalone token AND appears after at least one positional alias name. Guard against treating a literal `+` that is a flag value: only split if there is at least one non-flag, non-`+` token before the first `+`. Helper:
   ```ts
   const plusIdx = userArgv.indexOf('+');
   const hasComposition = plusIdx > 0
     && userArgv.slice(0, plusIdx).some((t) => !t.startsWith('-') && t !== '+');
   ```
   If `!hasComposition`, fall through to the existing `program.parseAsync` path unchanged.

3. When composing: extract a top-level `--parallel` flag and REMOVE it from the token stream so it is never passed to a segment:
   ```ts
   const parallel = userArgv.includes('--parallel');
   const tokens = userArgv.filter((t) => t !== '--parallel');
   ```
   (Re-derive `plusIdx`/segments from `tokens`, not the original.)

4. Split `tokens` on `+` into segments. Each segment is `[aliasName, ...extraArgs]`. Trim empty segments; if any segment is empty (e.g. `a + + b` or trailing `+`) emit a clear error and return 50.
   ```ts
   const segments: string[][] = [];
   let cur: string[] = [];
   for (const t of tokens) {
     if (t === '+') { segments.push(cur); cur = []; }
     else cur.push(t);
   }
   segments.push(cur);
   ```
   Validate: every segment has length >= 1.

5. Resolve run-level flags ONCE from the whole token stream so they apply to all segments (matches single-alias semantics where these are global to the invocation): detect `--dry-run`, `--verbose`, `--log`, `--ui`, `--list` as presence flags, and `--short-log <N>` / `--from <step>` by scanning tokens. Strip these run-level flags from each segment's extraArgs so they are not forwarded as pass-through args. Reuse the SAME flag names/logic already present in the action handler (XCI_FLAGS set, --short-log/--from extraction) â€” factor a tiny local helper if helpful, but keep it simple. The first token of each segment is the alias name; the remaining are that segment's extraArgs (after stripping run-level flags).

6. EARLY VALIDATION (edge case â€” fail before any execution): for each segment, if `!commands.has(aliasName)`, print the standard unknown-alias error and return 50 BEFORE running anything:
   ```
   error [CLI_UNKNOWN_ALIAS]: Unknown alias: "<name>"
     suggestion: Run `xci --list` to see available aliases
   ```

7. Execute:
   - Sequential (default): loop segments in order, `const code = await runAlias(alias, def, extraArgs, commands, config, projectRoot, flags); if (code !== 0) return code;` â€” stop on first non-zero, return that code. If all succeed return 0.
   - Parallel (`--parallel`): `const codes = await Promise.all(segments.map(seg => runAlias(...)));` then `return codes.find((c) => c !== 0) ?? 0;` â€” wait for all, return the FIRST non-zero (by segment order), else 0.

8. Return the computed exit code from `main` directly (do not fall through to `program.parseAsync`). The entry-point wrapper `main(...).then(code => process.exit(code))` propagates it.

Place this block so it short-circuits the normal flow: when `hasComposition` is true, run the multi-alias path and `return` its exit code; otherwise continue to the existing `program.action(...)` + `program.parseAsync` logic untouched.

Do NOT alter the `--agent` or `--get-completions` pre-scans at the top of main(). Do NOT change cold-start for the common single-alias path (the `+` check is a cheap `indexOf`).
