<objective>
Add a new DSL command kind `uproject` to the xci CLI (packages/xci) that edits Unreal Engine `.uproject` files (JSON) and manages plugins. The kind mirrors the existing `ini` kind end-to-end: it is a file-manipulation kind (not a process-spawn kind), detected by the presence of an `uproject` key, with the same cwd / `${var}` interpolation / params handling.

Purpose: lets users toggle UE plugins and edit `.uproject` fields declaratively from a versioned alias, exactly the way `ini` edits config files.
Output: a fully wired `uproject` kind — types, normalize, resolver, a new `executor/uproject.ts` with a PURE testable core, executor dispatch, sequential-step support, cwd resolution, help/list/dry-run/verbose output, TUI labels, unit + e2e tests, and a UE example alias.
</objective>
