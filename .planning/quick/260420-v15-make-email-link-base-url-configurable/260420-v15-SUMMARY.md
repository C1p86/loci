# Quick 260420-v15: Make email link base URL configurable — Summary

Introduce optional `APP_BASE_URL` env var so transactional email links (verify-email, password-reset, org invite) use a configurable canonical prefix instead of `https://${req.headers.host}`. Unblocks the `docker compose up` dev stack (emails now carry clickable `http://localhost:3000/...` links via MailHog) while preserving backward-compatible `https://<Host-header>` fallback for existing reverse-proxy prod deployments.

## What Changed

### `packages/server/src/config/env.schema.ts`

Added `APP_BASE_URL` to `properties` as optional (no entry in `required[]`, pattern `^https?://`) and to the `declare module 'fastify'` config type as `APP_BASE_URL?: string`. No default value — absence is semantically meaningful (falls back to Host-header construction). Comments reference Quick 260420-v15 and the backward-compat rationale.

### `packages/server/src/email/link.ts` (new)

Pure ~15-LOC helper exporting `buildEmailLink(ctx, path, queryKey, queryValue)`. Implementation:

```typescript
export function buildEmailLink(ctx, path, queryKey, queryValue) {
  const base = ctx.appBaseUrl ?? `https://${ctx.headerHost ?? 'localhost'}`;
  return `${base}${path}?${queryKey}=${encodeURIComponent(queryValue)}`;
}
```

No defensive validation (caller owns path-segment encoding). Helper is pure: no fastify, no DB, no logger — trivially unit-testable.

### `packages/server/src/email/__tests__/link.test.ts` (new)

5 vitest cases covering every branch of the fallback chain:

1. APP_BASE_URL set → prefix wins verbatim (headerHost ignored)
2. APP_BASE_URL unset, headerHost set → `https://<headerHost>` fallback
3. APP_BASE_URL unset, headerHost unset → `https://localhost` final fallback
4. URL-encoding applied to `queryValue` (tokens with `=` and `/`)
5. Pre-encoded path segment passes through unmodified (caller-owned)

### `packages/server/src/routes/auth/signup.ts`

Imports `buildEmailLink`. Line 42 template literal replaced with helper call passing `fastify.config.APP_BASE_URL` and `req.headers.host` as context.

### `packages/server/src/routes/auth/request-reset.ts`

Same as signup.ts — helper call with `/reset` path and `pr.token` token.

### `packages/server/src/routes/orgs/invites.ts`

Does NOT use the helper — the invite URL shape is path-segment-only (`/invites/<encoded-token>/accept`, NO query string). Inline form preserves that exact contract:

```typescript
const base = fastify.config.APP_BASE_URL ?? `https://${req.headers.host ?? 'localhost'}`;
const link = `${base}/invites/${encodeURIComponent(created.token)}/accept`;
```

Per plan instruction — a second helper variant was considered and explicitly rejected. Two call sites use the helper, one uses the inline expression; both respect the same fallback chain.

### `docker-compose.yml`

Added `APP_BASE_URL: http://localhost:3000` to the server service `environment:` block, alongside the existing SMTP topology overrides. This is what makes MailHog links clickable in local dev without further operator config.

### `.env.example`

Added commented `# APP_BASE_URL=http://localhost:3000` entry in the `--- Optional overrides ---` section at bottom, with 3-line explanation of shape, fallback semantics, and prod guidance.
