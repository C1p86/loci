# Phase quick-260605-n67 Plan 01: Fix TypeScript Compilation Errors Summary

**One-liner:** Zero tsc errors achieved by fixing exactOptionalPropertyTypes, noUncheckedIndexedAccess, DOM-lib type, and execa overload issues across 21 files in packages/xci.

## Objective

Fixed all TypeScript compilation errors in packages/xci. `npx tsc --noEmit` now exits with code 0 and zero diagnostic lines. No runtime behaviour change — all fixes are type-level only.

## Root Causes Fixed

**1. exactOptionalPropertyTypes violations** — Object literals passed `prop: T | undefined` where the target declared `prop?: T`. Fixed with conditional spread pattern: `...(val !== undefined ? { prop: val } : {})`.

**2. noUncheckedIndexedAccess violations** — Array/map indexing returns `T | undefined`. Fixed with non-null assertions (`!`) where guarded by length checks, and `if (val === undefined) continue;` guards.

**3. SequentialStep union narrowing** — Code excluded `set` variant from the union but not `prompt` or `ini`. Fixed by extending all guards to include all non-cmd variants: `step.kind === 'set' || step.kind === 'prompt' || step.kind === 'ini'`.

**4. DOM lib missing** — `CloseEvent` referenced globally but tsconfig has `lib: ["ES2022"]` (no DOM). Fixed by importing `CloseEvent` from `ws` package.

**5. execa overload resolution** — `execa(cmd, args, { reject: false })` caused overload ambiguity. Fixed with explicit `Options` type object and `as unknown as ResultPromise` cast.

**6. tsup format array** — `['esm'] as const` creates `readonly ["esm"]` not assignable to mutable `Format[]`. Fixed with `as import('tsup').Format[]`.
